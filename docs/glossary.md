# 用語集 (Glossary)

このドキュメントは、`stockv2` プロジェクトで使われている主要な技術・金融用語について解説します。

## 機械学習・データサイエンス関連

| 用語 | 解説 |
| :--- | :--- |
| **LightGBM** | このプロジェクトのモデルの心臓部。Microsoftが開発した、高速かつ高性能な機械学習アルゴリズムの一種（勾配ブースティング）。決定木というモデルを多数組み合わせることで、複雑なパターンを学習します。 |
| **モデル (Model)** | データから特定のパターンを学習し、未知のデータに対して予測を行うプログラムのこと。このプロジェクトでは「10日後に株価が3%以上動くか」を予測するモデルを指します。 |
| **特徴量 (Feature)** | モデルが予測を行うための入力データのこと。株価の移動平均線、RSI、米国の株価指数など、予測の「判断材料」となる様々な情報が特徴量として使われます。 |
| **学習 (Training)** | 過去のデータ（特徴量と、その時の正解ラベル）をモデルに見せて、特徴量と正解の関係性を学ばせるプロセス。`make train` がこれにあたります。 |
| **予測 (Prediction)** | 学習済みのモデルに、最新の特徴量を入力して、未来の結果（10日後の株価トレンド）を推論させること。`make predict` がこれにあたります。 |
| **ハイパーパラメータ (Hyperparameter)** | モデルの学習方法を制御するための「設定値」のこと。`learning_rate`（学習率）や `n_estimators`（決定木の数）などがあり、これらの値によってモデルの性能が大きく変わります。 |
| **Optuna** | 最適なハイパーパラメータの組み合わせを自動的に探索（チューニング）してくれる便利なPythonライブラリ。`make train SEARCH_METHOD=optuna` で使われます。 |
| **ROC AUC** | モデルの性能評価指標の一つ。「上昇するケース」と「そうでないケース」をどれだけ上手に見分けられるか、というモデルの「識別能力」を0.5〜1.0の数値で示します。1.0に近いほど優秀です。 |
| **バックテスト (Backtesting)** | 作成したモデルが、過去のデータに対してどれだけのパフォーマンス（損益など）を上げたかをシミュレーションすること。モデルの信頼性を測る上で非常に重要です。 |
| **特徴量の重要度 (Feature Importance)** | モデルが予測を行う際に、どの特徴量を「重要視」したかを示す指標。どの情報が株価予測に効いているのかを分析する手がかりになります。 |

## 金融・テクニカル分析関連

| 用語 | 解説 |
| :--- | :--- |
| **銘柄 (Ticker)** | 個別の金融商品（株式など）を識別するためのコード。例: `7203.T` (トヨタ自動車), `AAPL` (アップル)。 |
| **テクニカル指標 (Technical Indicator)** | 過去の株価や出来高のパターンから、将来の値動きを予測するために用いられる指標。 |
| **SMA (単純移動平均線)** | ある一定期間の終値の平均値を線で結んだもの。トレンドの方向性を見るために使われます。 |
| **RSI (相対力指数)** | ある期間において、株価が「買われすぎ」か「売られすぎ」かを示す指標。0〜100%で示され、一般的に70%以上で買われすぎ、30%以下で売られすぎと判断されます。 |
| **MACD** | 2つの移動平均線の差を用いて、トレンドの転換点や勢いを見るための指標。 |
| **ボリンジャーバンド (Bollinger Bands)** | 移動平均線とその上下に標準偏差の線を加えたもの。価格がこのバンド内に収まることが多いという統計学的な性質を利用して、買われすぎや売られすぎを判断します。 |
| **VIX指数** | 「恐怖指数」とも呼ばれる、市場の不確実性や投資家の不安を示す指標。一般的に、VIXが上昇すると株価は下落する傾向があります。 |

## ソフトウェア・インフラ関連

| 用語 | 解説 |
| :--- | :--- |
| **Docker** | アプリケーションとその実行環境を「コンテナ」という単位にまとめて管理する技術。これにより、どんなPC上でも同じ環境を簡単に再現できます。 |
| **Makefile** | `make build` や `make train` のような短いコマンドを定義するためのファイル。複雑なコマンドライン操作を単純化し、プロジェクトの共通操作をまとめるために使われます。 |
| **PostgreSQL** | このプロジェクトで使われている、信頼性の高いオープンソースのデータベース管理システム。学習済みモデルや日々の予測結果などを永続的に保存します。 |
| **SSHトンネル** | インターネット上の安全でない経路を通って、暗号化された安全な通信路（トンネル）を確立する技術。このプロジェクトでは、外部のデータベースサーバーに安全に接続するために使われています。 |
| **CI/CD (継続的インテグレーション/継続的デリバリー)** | コードの変更をリポジトリにプッシュすると、テスト、ビルド、デプロイといった一連のプロセスが自動的に実行される仕組み。品質を維持し、リリースを高速化します。 |

## Pythonライブラリ関連

| 用語 | 解説 |
| :--- | :--- |
| **pandas** | データ分析を行うためのPythonライブラリ。DataFrameという表形式のデータ構造を使い、データの読み込み、加工、集計、フィルタリングなどを効率的に行います。このプロジェクトのデータ処理の中核です。 |
| **scikit-learn** | Pythonで最も有名な機械学習ライブラリ。モデルの学習、評価、データの前処理（`StandardScaler`など）といった、機械学習に必要な様々な機能を提供します。 |
| **psycopg2** | PythonからPostgreSQLデータベースに接続し、操作するためのライブラリ（アダプタ）。 |
| **yfinance** | Yahoo! Financeから株価や企業情報を手軽に取得できるPythonライブラリ。データ更新 (`update_stock_data.py`) で使われています。 |
| **pandas-ta** | `pandas` のDataFrameに、RSIやMACD、ボリンジャーバンドといった**100種類以上**のテクニカル分析指標を簡単に追加できる拡張ライブラリ。特徴量生成 (`create_features`) で活用されています。 |
| **gspread** | PythonからGoogleスプレッドシートを操作するためのライブラリ。通知システム (`send_notifications.py`) で、予測結果などをスプレッドシートに書き込むために使われます。 |

| **pytest** | Pythonのテストフレームワーク。コードが期待通りに動作するかを検証する「テストコード」を記述し、実行するために使います。 |
| **scikit-learn-intelex** | Intelが提供する、scikit-learnやLightGBMの処理を自動的に高速化するためのライブラリ。oneDAL (oneAPI Data Analytics Library) をバックエンドで利用します。 |

## プロジェクト固有の用語

| 用語 | 解説 |
| :--- | :--- |
| **`config.ini`** | プロジェクトの挙動を制御する様々な設定値（予測期間、閾値、テクニカル指標のパラメータなど）を、コードから分離して一元管理するためのファイル。 |
| **`target_tickers` (テーブル)** | 予測対象とする銘柄と、そのモデル学習時に特徴量として使用する関連銘柄（指数など）のリストを管理する、データベース上のテーブル。 |
| **`trained_models` (テーブル)** | 学習済みのLightGBMモデル本体、その性能評価（ROC AUCなど）、学習に使った特徴量のリストやハイパーパラメータなどをバージョン管理して保存する、データベース上のテーブル。「モデルレジストリ」とも呼ばれます。 |
| **`prediction_results` (テーブル)** | `make predict-all` によって日々行われる予測の結果（どの銘柄が、どのモデルで、何%の確率で上昇/下落するか）を時系列で蓄積していく、データベース上のテーブル。 |
